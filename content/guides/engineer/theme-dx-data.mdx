# DX・データ活用に使える経営工学キーワード

## 1. このテーマが技術士試験で重要になる理由

DXは単なるIT導入ではなく、業務プロセス、意思決定、顧客価値を変えるテーマです。経営工学部門では、データ基盤、標準化、KPI、人材育成、効果検証を組み合わせて論じる必要があります。

## 2. 社会背景・白書背景

人手不足、生産性向上、デジタル投資、データ連携、AI活用は、近年の産業政策や白書背景と接続しやすい論点です。一方で、データ品質、セキュリティ、説明責任、現場定着も課題になります。

## 3. このテーマで使える主要キーワード

| キーワード | 答案での使いどころ |
|---|---|
| [DX](/guides/engineer/dx) | 業務変革、顧客価値、データ活用を統合する |
| データドリブン | 経験依存からデータに基づく意思決定へ変える |
| ERP | 販売、生産、会計、在庫など基幹情報を統合する |
| MES | 工程実績、品質、設備状態を現場で取得する |
| WMS | 倉庫内在庫、入出庫、ロケーションを管理する |
| TMS | 配車、輸送進捗、積載率を管理する |
| EDI | 企業間の受発注・出荷情報を連携する |
| RFID | 個体識別、在庫精度、トレーサビリティを高める |
| デジタルツイン | 工程や物流を仮想空間で評価する |
| [KPIマネジメント](/guides/engineer/kpi-management) | DX効果を測定可能な指標で管理する |

## 4. 答案での使いどころ

課題では、部門ごとにデータが分断され、改善判断が属人的になっている状態を示します。解決策では、業務標準化、マスタ整備、システム連携、KPI設計、教育訓練を一体で進める構成にします。

## 5. 課題抽出に使える視点

- データが部門別に分断されている
- マスタや入力ルールが統一されていない
- システム導入が目的化している
- 現場がデータを活用できていない
- 投資効果を測るKPIがない

## 6. 解決策に使える視点

- 業務フローを標準化し、必要データを定義する
- ERP、MES、WMS、TMSを連携して、実績情報を一元化する
- KPIを設定し、改善活動と効果検証を結びつける
- 現場教育により、データを使った改善を定着させる

## 7. リスクと対策に使える視点

システム導入後に、入力負荷、データ不整合、現場抵抗、サイバーリスクが生じます。小さく試行し、データ定義、権限管理、教育、運用レビューを行うことが対策になります。

## 8. 技術者倫理・社会の持続可能性への接続

データ活用では、個人情報、機密情報、説明責任、公平性が重要です。技術者は、効率化だけでなく、人間中心、透明性、セキュリティ、継続的な能力向上を踏まえてDXを進めます。

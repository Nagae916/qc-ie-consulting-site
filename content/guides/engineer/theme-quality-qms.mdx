# QMS再構築・品質不正防止に使える経営工学キーワード

## 1. このテーマが技術士試験で重要になる理由

品質問題は、現場の不注意だけでなく、プロセス保証、記録、監査、ガバナンス、倫理が弱いと発生します。技術士答案では、品質管理手法とQMSを経営の仕組みとして接続することが重要です。

## 2. 社会背景・白書背景

品質不正、検査データ改ざん、サプライチェーン複雑化、顧客要求高度化が背景です。社会的信頼、説明責任、技術者倫理、持続可能な品質保証と接続できます。

## 3. このテーマで使える主要キーワード

| キーワード | 答案での使いどころ |
|---|---|
| [QMS再構築](/guides/engineer/qms-reconstruction) | 品質保証を文書管理から実効的な仕組みに変える |
| [KPIマネジメント](/guides/engineer/kpi-management) | 品質KPIを設定し、改善を継続管理する |
| BSC | 品質、顧客、プロセス、人材をバランスよく評価する |
| MES | 工程実績と品質データを記録する |
| トレーサビリティ | 原因追跡、説明責任、再発防止に使う |
| データドリブン | 品質判断を経験依存からデータ基盤へ移す |
| 技術者倫理 | 品質不正防止、公益確保、説明責任を示す |
| CSR | 顧客、社会、取引先への責任を説明する |
| ガバナンス | 組織として不正を防ぐ管理体制を作る |

## 4. 答案での使いどころ

課題では、文書中心のQMS、現場との乖離、品質データの未活用、監査の形骸化を挙げます。解決策では、プロセスアプローチ、FMEA、CAPA、品質KPI、内部監査、教育訓練を組み合わせます。

## 5. 課題抽出に使える視点

- QMSが認証維持のための文書管理に偏っている
- 品質データが工程改善に活用されていない
- 不適合の真因分析と再発防止が弱い
- 監査が形式確認で終わっている
- 品質倫理とガバナンスが現場に浸透していない

## 6. 解決策に使える視点

- 業務フローを可視化し、重要工程を明確にする
- FMEAとCAPAを連動させる
- 不良率、工程能力、再発率など品質KPIを設定する
- MESやトレーサビリティで記録と証拠を確保する
- 内部監査を改善提案型に変える

## 7. リスクと対策に使える視点

QMS再構築では、記録負荷、形式運用化、現場抵抗、データ品質低下がリスクになります。対策として、入力項目を絞り、教育訓練、監査員育成、KPIレビューを行います。

## 8. 技術者倫理・社会の持続可能性への接続

品質不正防止は公益確保そのものです。技術者は、短期の納期やコストを優先して品質リスクを隠さず、顧客安全、法令順守、説明責任を満たす仕組みを設計します。

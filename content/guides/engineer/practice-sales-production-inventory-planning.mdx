# 販売・生産・在庫計画連携｜技術士 経営工学 Ⅱ-2型答案練習

## テーマを確認する

販売・生産・在庫計画連携は、販売部門、生産管理部門、製造部門、購買部門、物流部門が別々に計画を立てる状態を改善し、需要、能力、在庫、調達、出荷を整合させる活動です。

S&OPやPSIを使い、需要予測、販売計画、生産計画、在庫計画を共通データと共通KPIで結びます。ERPやMRPを活用する場合も、マスターデータ、意思決定ルール、部門間連携が弱いと計画は形骸化します。

## 想定設問

販売計画、生産計画、在庫計画が部門ごとに分断され、欠品、過剰在庫、生産計画変更、納期遅延が発生している。このような状況で、販売・生産・在庫計画の連携を改善する業務を担当する場合、どのように進めるか。調査・検討すべき事項、業務の進め方、留意点および工夫点を述べよ。

選択科目Ⅱ-2型を想定し、1200字以内で解答します。

## 問題文を読み取る

この設問で求められていること：

- 部門別計画の分断を解消すること
- 販売計画、生産計画、在庫計画を共通前提で結ぶこと
- S&OPやPSIで定期的に需給を調整すること
- ERP/MRPなどのデータ基盤を活用すること
- 部門最適、データ品質、計画形骸化に留意すること

答案で外してはいけない点：

- 販売、生産、在庫の計画連携
- 共通KPIと意思決定ルール
- ERP/MRPやマスターデータの整備
- 経営層を含む合意形成
- 欠品、過剰在庫、納期遅延の効果確認

## 使うキーワードを選ぶ

主要キーワード：

- S&OP
- PSI
- SCM
- ERP
- MRP
- 需要予測
- 在庫最適化
- 共通KPI
- 部門間連携
- データガバナンス
- 納期遵守率
- 欠品率

| キーワード | 使いどころ |
|---|---|
| S&OP | 販売・生産・在庫を経営判断で整合させる |
| PSI | 生産・販売・在庫を数量で整合させる |
| SCM | 調達から販売まで全体最適化する |
| ERP | 部門横断のデータ基盤 |
| MRP | 資材所要量と生産計画を整合させる |
| 需要予測 | 販売計画の前提 |
| 共通KPI | 部門間の目標をそろえる |
| データガバナンス | 計画データの信頼性を高める |

## 答案骨子を作る

### 調査、検討事項

- 販売計画、生産計画、在庫計画の作成方法を確認する
- 計画更新頻度、使用データ、責任部門、承認プロセスを確認する
- 欠品率、在庫回転率、計画変更回数、納期遵守率を確認する
- ERP/MRPのマスタデータ、需要予測、在庫基準を確認する
- 部門間会議や意思決定ルールを確認する

### 分析、方針設定

- 計画差異の要因を需要変動、能力制約、在庫基準、データ不整合に分ける
- 販売、生産、在庫で共通KPIを設定する
- S&OPまたはPSIの会議体と意思決定プロセスを設計する
- 重点品目、重点顧客、制約工程から改善対象を選定する

### 実施手順

- 計画データとマスタデータを整備する
- 需要予測と販売計画を統合する
- 生産能力、調達制約、在庫水準を踏まえて計画を調整する
- S&OPやPSI会議を定期開催する
- ERP/MRPへ確定計画を反映する
- 試行後、対象品目や拠点を段階的に拡大する

### 留意点、工夫点

- 会議体を作るだけで終わらせない
- 部門最適のKPIを見直す
- データ品質を確保する
- 計画変更のルールを明確にする
- 販売機会損失と在庫増加のバランスを見る

### 関係者調整（サプライチェーン全体）

- 販売、営業、製造、生産管理、購買、物流、経営層で需給方針を共有する
- サプライヤーと調達制約や納入頻度を調整する
- 顧客要求納期と生産能力を整合させる
- 関係者が同じデータを見て判断できる体制を作る

### 効果確認

- 欠品率、在庫回転率、納期遵守率、計画変更回数、予測誤差を確認する
- S&OPやPSI会議で計画と実績の差異を確認する
- KPIを定期的に見直し、PDCAで改善する

## 1200字以内で答案を書く

答案例：約1100字

販売・生産・在庫計画連携を改善する際は、各部門が個別に計画を作る状態を改め、需要、能力、在庫、調達、出荷を共通前提で整合させる必要がある。まず調査・検討事項として、販売計画、生産計画、在庫計画の作成方法、更新頻度、使用データ、責任部門、承認プロセスを確認する。あわせて、欠品率、在庫回転率、計画変更回数、納期遵守率、予測誤差を確認し、ERP/MRPの品目マスタ、BOM、リードタイム、在庫基準の整備状況も把握する。

次に、計画差異の要因を需要変動、販売計画の精度、生産能力制約、調達制約、在庫基準、データ不整合に分けて分析する。重点品目、重点顧客、制約工程を選定し、販売、生産、在庫で共通KPIを設定する。KPIは欠品率、在庫回転率、納期遵守率、計画変更回数、予測誤差とし、部門別の売上、稼働率、在庫削減だけで判断しない方針を明確にする。短期の需給調整と中期の能力・調達制約の見直しを分けて扱う。特に欠品影響が大きい品目を優先する。

実施手順として、まず計画データとマスタデータを整備し、品目コード、リードタイム、ロット、在庫基準を統一する。次に需要予測と販売計画を統合し、生産能力、調達制約、在庫水準を踏まえてPSIを作成する。S&OP会議を定期開催し、販売、生産管理、購買、物流、経営層が同じデータで需給を確認する。確定した計画はERP/MRPに反映し、資材所要量、生産指示、在庫補充へつなげる。計画変更が必要な場合は、変更期限、承認者、影響確認項目を決める。まず重点品目で試行し、会議運営とデータ精度を確認してから対象品目や拠点を拡大する。

留意点として、会議体を作るだけでは連携は改善しない。意思決定者、締切、変更ルール、例外対応を明確にし、計画が形骸化しないようにする。また、販売機会損失を恐れて過大な販売計画にすると過剰在庫を招き、在庫削減だけを優先すると欠品を招く。データ品質が低い場合は、現場判断と照合しながら段階的に精度を高める。計画変更の頻発も管理対象にし、変更理由を記録する。例外処理も共有して必ず記録として残す。

関係者調整として、販売部門とは需要見込みや顧客要求納期を共有し、製造・生産管理とは能力制約と生産順序を調整する。購買部門とは調達リードタイムやサプライヤー制約を確認し、物流部門とは出荷能力と保管スペースを確認する。経営層には需要、在庫、能力のトレードオフを示し、方針判断を得る。改善後は欠品率、在庫回転率、納期遵守率、計画変更回数、予測誤差を月次で確認し、S&OPで計画と実績の差異を振り返る。緊急変更や特急手配の件数も確認し、販売・生産・在庫計画連携を継続的に改善する。

## 留意点と工夫点を確認する

留意点：

- 会議体を作るだけで終わらせない
- 部門ごとのKPIが対立しないようにする
- マスターデータや計画データの品質を確認する
- 計画変更のルールを曖昧にしない
- 販売機会損失と在庫増加のバランスを見る

工夫点：

- S&OPやPSIで共通データを使う
- 共通KPIで販売・生産・在庫を結ぶ
- ERP/MRPへ確定計画を反映する
- 重点品目から試行する
- 経営層を含めて意思決定ルールを明確にする

## 自己チェックする

- 販売計画、生産計画、在庫計画の作成方法を確認しているか
- 計画差異を需要、能力、在庫、データ不整合に分けているか
- S&OP、PSI、ERP、MRPを適切に使っているか
- 共通KPIと意思決定ルールがあるか
- 販売、製造、生産管理、購買、物流、経営層との調整があるか
- 効果確認で欠品率、在庫回転率、納期遵守率、計画変更回数を使っているか
- 1200字以内で、設問の要求に過不足なく答えているか

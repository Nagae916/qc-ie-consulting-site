# 技術士コンピテンシー参照メモ

## 目的

このメモは、技術士第二次試験コンテンツの内部設計メモです。修習ガイドブック第3.1版および技術士に求められる資質能力を、答案骨子、例題生成、過去問トレンド分析に反映するための要約として整理します。

原資料の文章は転載せず、サイト内の自己確認・設計データとして利用します。公開ページで扱う場合は、出典リンクを明示します。

## 修習ガイドブック第3.1版を参照する目的

修習ガイドブックは、技術士を目指す技術者が、実務経験を通じて必要な資質能力を計画的に身につけるための基盤資料として扱う。本サイトでは、二次試験の答案を単なる過去問対策ではなく、技術士に必要な資質能力を答案上で表現する訓練として設計する。

## IPDとは何か

IPDは、技術者が技術士に必要な能力を実務や修習を通じて段階的に獲得するための初期専門能力開発として扱う。ここでは、単発の知識学習ではなく、実務課題を振り返り、専門知識、問題解決、マネジメント、倫理などを統合して伸ばすプロセスとして位置づける。

## 8つのコンピテンシー

- 専門的学識
- 問題解決
- マネジメント
- 評価
- コミュニケーション
- リーダーシップ
- 技術者倫理
- 継続研さん

## 答案上の見え方

| コンピテンシー | 答案上の見え方 |
| --- | --- |
| 専門的学識 | 経営工学、品質管理、生産管理、統計、法令・制度を、課題に応じて適切に使い分ける。 |
| 問題解決 | 問題点、技術課題、最重要課題、解決策、リスクが一貫している。 |
| マネジメント | 人、設備、情報、費用、納期、品質などの資源や制約を踏まえて施策を組み立てる。 |
| 評価 | KPI、効果確認、波及効果、施策後リスクを明示する。 |
| コミュニケーション | 関係者、説明責任、合意形成、情報共有の方法を具体化する。 |
| リーダーシップ | 部門横断の調整、推進体制、役割分担、現場への定着を示す。 |
| 技術者倫理 | 安全、公益、法令順守、情報管理、不正防止、持続可能性をテーマ固有に書く。 |
| 継続研さん | 新技術、法令、標準、白書、現場知見を継続的に学び改善へ反映する姿勢を示す。 |

## 試験科目への対応

| 科目 | 重点 |
| --- | --- |
| 必須科目Ⅰ | 問題解決、評価、コミュニケーション、技術者倫理を中心に、専門的学識とマネジメントも答案全体で表現する。 |
| 選択科目Ⅱ-1 | 専門的学識を中心に、用語・手法の目的、適用場面、留意点を簡潔に示す。 |
| 選択科目Ⅱ-2 | 専門的学識、マネジメント、コミュニケーションを中心に、手順、関係者調整、留意点を整理する。 |
| 選択科目Ⅲ | 問題解決、評価、リーダーシップ、技術者倫理を中心に、専門領域の課題と将来展望を示す。 |

## AnswerStructureBuilderへの反映方針

- 出力下部に「コンピテンシー確認」パネルを追加する。
- 採点や点数化はしない。
- 各コンピテンシーの説明と、答案上の良い表現の兆候をチェックリストとして表示する。
- ユーザーが、答案骨子に不足している観点を自己確認できる状態にする。

## PastExamTrendMapへの反映方針

- 過去問メタデータに `assessedCompetencies` と `competencyNotes` を付与する。
- すべての問題に完全付与できない場合は、必須科目Ⅰ型を優先する。
- 問題カードに「この問題で意識するコンピテンシー」を表示する。
- 表示名は `competencies.json` の `label` を使う。

## 例題生成への反映方針

- 必須Ⅰ型例題生成MVPでは、標準セットとして以下を固定表示する。
  - 専門的学識
  - 問題解決
  - 評価
  - コミュニケーション
  - 技術者倫理
- テーマによって、マネジメント、リーダーシップ、継続研さんも追加できる構造にする。

## 今回の範囲

今回は、採点・点数化・AI添削・合否判定は行わない。コンピテンシーは、答案骨子や例題を確認するための自己確認チェックとして使う。

# n-ie-qclab 情報設計・ページ役割ルール

## 1. サイトの目的

n-ie-qclab は、経営工学を中心に、QC・品質管理、統計・データ分析、技術士 経営工学部門、生産管理をつなぎ、試験対策と実務改善の両方に使える学習サイトである。

このサイトでは、単に用語を読むだけでなく、次の流れを支援する。

1. 知識を読む
2. 図解・インタラクティブ教材で理解する
3. 問題・答案で使う
4. 実務改善や試験答案に転用する

## 2. 4本柱

| 柱 | 役割 |
|---|---|
| QC・品質管理 | 品質保証、品質改善、QMS、QC検定、管理図、工程能力、実験計画法などを学ぶ |
| 統計・データ分析 | 記述統計、確率分布、推定、検定、回帰分析、分散分析などを学ぶ |
| 技術士 経営工学部門 | 過去問分析、設問形式、重要キーワード、答案骨子、答案練習を通じて試験対策を行う |
| 生産管理 | 生産計画、工程管理、在庫管理、IE、設備効率、SCM、物流管理を学ぶ |

## 3. ページ種別

| ページ種別 | 役割 | 入れてよいもの | 入れないもの |
|---|---|---|---|
| 入口ページ | 学習の始点を示す | 学習順、主要リンク、全体像 | 詳細な分析表、長い解説、演習UI |
| マップページ | 全体像・分類・優先順位を示す | 俯瞰表、分類、優先順位、次に読むページ | 問題生成、詳細フィルタ、長い入力UI |
| 個別解説ページ | 1テーマを深く学ぶ | 定義、特徴、課題、対策、例、答案での使い方 | サイト全体説明、大量の一覧 |
| 演習ページ | 問題を解く・答案骨子を作る | 問題、テーマ選択、骨子、回答例、入力欄 | サイト全体説明、過剰な理論説明 |
| 参照ページ | 白書・過去問・用語などを確認する | 出典、要約、一覧、参照リンク | 演習UI、長い入力欄 |

## 4. ページ作成前チェックリスト

新しいページを作る前に、必ず以下を確認する。

- ページの主目的は1つか
- ページ種別は何か
- このページでやることは何か
- このページでやらないことは何か
- 入口リンクはどこか
- 出口リンクはどこか
- 既存ページと役割が重複していないか
- 演習UIをマップページに混ぜていないか
- 詳細分析を入口ページに混ぜていないか
- ユーザーが次に何をすればよいか分かるか

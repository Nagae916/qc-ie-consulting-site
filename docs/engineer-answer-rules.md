# 技術士二次試験 答案骨子作成ルール

## 1. 基本方針

AnswerStructureBuilder は、一般的な作文テンプレートではなく、技術士第二次試験の設問要求に沿って、課題抽出、最重要課題、解決策、リスク、技術者倫理、社会の持続可能性を構造化するための教材として設計する。

答案は「試験で読まれる文章」であると同時に、「技術者として実務課題をどう捉え、どう解決するか」を示す文章である。そのため、抽象的な言い換えではなく、テーマ固有の具体語、現場で起こる問題、施策実施後の影響までを意識して組み立てる。

3枚答案を想定し、設問ごとの配分を意識する。各設問で同じ内容を繰り返さず、問題文の要求に対して不足なく、過剰なく答える。

## 2. 問題文の要求整理

答案作成前に、問題文から以下を整理する。

- 対象となる社会・産業・組織・現場
- 求められている立場
- 問われているテーマ
- 抽出すべき課題の範囲
- 解決策を示す対象
- リスクを考えるタイミング
- 技術者倫理・社会の持続可能性で触れるべき観点

問題文の要求を読み違えると、どれだけ文章が整っていても評価されにくい。まず「何を答える問題か」を明確にする。

## 3. 設問(1) 課題抽出のルール

設問(1)では、課題を3つ明確に示す。

課題は抽象語で終わらせない。「人材不足」「コスト増」「品質向上が必要」だけでは不十分であり、何が、どの場面で、なぜ問題になっているかを具体化する。

基本の流れは、問題点から課題へつなげる。

- 問題点: 現状で起きている困りごと、阻害要因、放置した場合の影響
- 課題: 技術者として取り組むべき方向性

問題点と課題を混同しない。問題点は「悪い状態」、課題は「改善すべき方向」である。

課題3つは、可能な限り観点を分ける。例として、人材・組織、業務プロセス、情報・データ、設備・現場、品質、サプライチェーン、経営、社会・制度などがある。

## 4. 設問(2) 最重要課題と解決策のルール

最重要課題は、設問(1)で抽出した課題の中から選ぶ。設問(1)にない課題を突然、最重要課題として出さない。

最重要課題を選ぶ理由は、以下の観点から説明する。

- 影響範囲が広い
- 根本原因に近い
- 他課題への波及効果が大きい
- 緊急性が高い
- 実現可能性が高い

解決策は、最重要課題に直接対応させる。課題と解決策がずれていると、答案全体の一貫性が弱くなる。

解決策は、できるだけ複数の施策に分けて書く。施策は単なるスローガンではなく、実行主体、対象、方法、期待効果が伝わるようにする。

## 5. 設問(3) リスクと対策のルール

リスクは、施策実施後に新たに生じるものを書く。

すでに設問(1)で挙げた問題点や課題を、リスクとして繰り返さない。リスクは「解決策を実行した結果、別の副作用として起こり得ること」である。

リスクの例は以下のように整理する。

- デジタル化に伴う情報セキュリティリスク
- 標準化に伴う現場裁量の低下
- 自動化に伴う技能継承機会の減少
- データ活用に伴う誤判定や責任分界の不明確化
- サプライチェーン連携に伴う情報管理負荷の増加

リスク対策は、リスクに直接対応させる。監視、教育、ルール化、権限設計、段階導入、レビュー体制など、実行可能な対策にする。

## 6. 設問(4) 技術者倫理・社会の持続可能性のルール

技術者倫理・社会の持続可能性は、一般論で終わらせない。

「法令遵守」「安全確保」「環境配慮」「社会貢献」だけでは弱い。テーマ固有の具体語を使い、何を守り、誰にどのような影響があるかを示す。

製造業DXであれば、以下のような具体語を使う。

- 品質保証
- 顧客安全
- 情報セキュリティ
- 作業者の技能と雇用
- サプライチェーンの安定
- 中小企業との取引適正化
- データガバナンス

社会の持続可能性では、短期的な効率だけでなく、長期的な品質、働きがい、人材育成、環境負荷、地域産業への影響も考慮する。

## 7. 添削フィードバックからの再発防止ルール

添削で指摘されやすい点を、作成時のチェック項目として扱う。

- 課題が抽象的で、何を改善するのか分からない
- 課題3つの観点が重複している
- 問題点、課題、解決策が混在している
- 最重要課題が設問(1)の課題と対応していない
- 解決策が一般論で、最重要課題への対応になっていない
- リスクが施策実施後の新たなリスクになっていない
- 技術者倫理・持続可能性が汎用語だけで終わっている
- 3枚答案の配分が崩れ、設問ごとの厚みが偏っている

AnswerStructureBuilder では、これらの再発防止に役立つ入力欄、確認文、補助表示を設ける。

## 8. AnswerStructureBuilder に反映する入力項目

AnswerStructureBuilder では、最低限以下の入力項目を扱う。

- テーマ
- 本番風の設問文
- 問題文から読み取った要求
- 課題1
- 課題2
- 課題3
- 各課題の問題点
- 各課題の観点
- 最重要課題
- 最重要課題を選ぶ理由
- 解決策
- 解決策の実行主体
- 解決策の期待効果
- 施策実施後に生じるリスク
- リスク対策
- 技術者倫理で触れる具体語
- 社会の持続可能性で触れる具体語
- 600字答案用の短い骨子
- 1800字答案用の詳細骨子

## 9. 今回は実装しない機能

以下は今後の拡張候補とし、初期実装では扱わない。

- AI添削
- 自動採点
- 難易度設定
- タイマー
- localStorage への保存
- ユーザー別の学習履歴
- ランダム問題生成
- 添削講座テキストの自動取り込み
- 公的資料の自動更新
- PDF出力

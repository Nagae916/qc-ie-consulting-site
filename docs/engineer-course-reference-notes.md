## 1. 技術士第二次試験の科目構成

経営工学部門の第二次試験では、必須科目Ⅰと選択科目Ⅱ・Ⅲで求められる答案型が異なる。

- 必須科目Ⅰ：部門全体に関わる社会的課題、技術者としての課題抽出、解決策、リスク、倫理・持続可能性を問う
- 選択科目Ⅱ-1：専門用語、手法、特徴、適用場面、留意点を短く説明する
- 選択科目Ⅱ-2：具体的な業務・プロジェクトを想定し、調査事項、実施手順、留意点、関係者調整を説明する
- 選択科目Ⅲ：専門分野に関する中長期的課題、解決策、リスク、将来展望を総合的に論述する

AnswerStructureBuilder と PastExamTrendMap では、この科目構成ごとに答案フレームを分ける。

## 2. 必須科目Ⅰの出題意図と答案構成

必須科目Ⅰは、経営工学部門の技術者として、社会・産業・組織の変化を多面的に捉え、課題と解決策を構造化できるかを確認する問題である。

答案構成の基本は以下とする。

1. 問題文の背景を読み取り、対象と立場を明確にする
2. 多面的な観点から課題を3つ抽出する
3. 3課題の中から最重要課題を1つ選ぶ
4. 最重要課題に直接対応する解決策を示す
5. 施策実施後に新たに生じるリスクと対策を示す
6. 技術者倫理と社会の持続可能性を、テーマ固有の具体語で述べる

必須科目Ⅰでは、課題抽出、最重要課題、解決策、リスク、倫理・持続可能性の一貫性を最重視する。

## 3. 選択科目Ⅱ-1の出題意図と答案構成

選択科目Ⅱ-1は、専門用語や経営工学手法を正確に理解し、短い答案で説明できるかを確認する問題である。

答案構成の基本は以下とする。

1. 用語・手法の定義を示す
2. 目的や特徴を説明する
3. 適用場面を示す
4. 使用時の留意点や限界を述べる
5. 必要に応じて、品質・生産・サービスの実務例に接続する

単なる暗記説明ではなく、実務でどう使うか、どのような注意点があるかまで含める。

## 4. 選択科目Ⅱ-2の出題意図と答案構成

選択科目Ⅱ-2は、具体的な業務場面で、技術者として実施手順や留意点を整理できるかを確認する問題である。

答案構成の基本は以下とする。

1. 業務の目的と対象範囲を整理する
2. 調査・検討すべき事項を挙げる
3. 実施手順を順序立てて説明する
4. 関係者、制約条件、品質・コスト・納期への影響を考慮する
5. 実施上の留意点、リスク、対策を述べる

選択科目Ⅱ-2では、手順が抽象的にならないようにし、誰が、何を、どの順番で、どの指標を見て進めるかを意識する。

## 5. 選択科目Ⅲの出題意図と答案構成

選択科目Ⅲは、専門分野の中長期的な課題を捉え、解決策、リスク、将来展望までを論述できるかを確認する問題である。

答案構成の基本は以下とする。

1. 背景となる社会・産業・技術動向を整理する
2. 専門分野の観点から課題を複数抽出する
3. 重要課題に対する解決策を示す
4. 実施上のリスクや制約を整理する
5. 今後の展望、持続可能性、技術者としての役割を述べる

選択科目Ⅲでは、単なる現状説明ではなく、将来変化に対して経営工学としてどう対応するかを示す。

## 6. 経営工学部門で重視される観点

経営工学部門では、以下の観点を組み合わせて課題を整理する。

- QCD
- 生産性
- 品質管理
- 生産・物流マネジメント
- サービスマネジメント
- 業務プロセス
- 人材・組織
- データ活用・DX
- サプライチェーン
- リスク管理
- コスト・投資対効果
- 顧客価値
- 標準化・教育訓練
- 持続可能性

答案では、これらを単語として並べるだけでなく、対象業務の問題点、課題、解決策、評価指標に接続する。

## 7. 答案作成時の注意点

答案作成時は以下を避ける。

- 問題文の要求を読み替えてしまう
- 課題が抽象語だけで終わる
- 問題点、課題、解決策が混在する
- 設問(1)で出していない課題を、設問(2)の最重要課題にする
- 解決策が最重要課題に対応していない
- リスクが現状課題の繰り返しになっている
- 倫理・持続可能性が一般論だけで終わる
- 経営工学らしい手法や評価指標が不足する

答案では、問題点 → 技術課題 → 最重要課題 → 解決策 → 施策後リスク → 倫理・持続可能性の流れを守る。

## 8. 解答例に共通する章立て

解答例に近い章立てとして、以下を基本形にする。

### 必須科目Ⅰ型

1. 課題
2. 最重要課題と解決策
3. リスクと対策
4. 技術者倫理・社会の持続可能性

### 選択科目Ⅱ-1型

1. 定義
2. 特徴・目的
3. 適用場面
4. 留意点

### 選択科目Ⅱ-2型

1. 調査・検討事項
2. 実施手順
3. 留意点・工夫点
4. 関係者調整・リスク対応

### 選択科目Ⅲ型

1. 背景と課題
2. 解決策
3. リスク・制約
4. 今後の展望

## 9. 添削課題で再発防止すべき指摘

添削課題の観点から、以下を再発防止ルールとして扱う。

- 設問要求に対して答えていない
- 課題の粒度がそろっていない
- 抽象語が多く、何を改善するのか分からない
- 経営工学手法や評価指標が不足している
- 最重要課題の選定理由が弱い
- 解決策が一般論で、実行主体や管理指標がない
- リスクと対策が一対一で対応していない
- 倫理・持続可能性がテーマ固有の表現になっていない

これらは AnswerStructureBuilder の確認ポイントや、例題生成時の骨子ガイドに反映する。

## 10. 各機能に反映するルール

### AnswerStructureBuilder

- 科目ごとに答案型を分ける
- 問題文の要求整理を最初に置く
- 課題3つ、最重要課題、解決策、リスク、倫理を一貫して入力させる
- 出力形式は、構造整理シート、答案骨子、600字メモに分ける
- 確認ポイントとして、抽象語、課題と解決策の対応、リスクの性質を表示する

### PastExamTrendMap

- 過去問本文ではなく、要約、設問パターン、答案フレーム、タグを扱う
- 必須科目Ⅰ、選択科目Ⅱ-1、Ⅱ-2、Ⅲで答案型を分けて表示する
- 頻出タグだけでなく、答案型と学習アクションを表示する
- 傾向分析は、講座資料で重視される答案構成と矛盾しないようにする

### 例題生成

- 公式問題文を転載しない
- 過去問メタデータから、テーマ、政策、法令、設問構造を参照してオリジナル例題を作る
- 必須科目Ⅰ型では、課題抽出、最重要課題、解決策、リスク、倫理・持続可能性を必ず含める
- 生成した例題には、参考にしたメタデータと骨子作成ガイドを表示する

# n-ie-qclab サイト方針

## サイトの位置づけ

n-ie-qclab は、品質管理単独サイトでも統計単独サイトでもなく、「経営工学」を中心軸にした実務型学習サイトとする。

## 基本コンセプト

経営工学を、試験で終わらせず、現場で使える力に変える。

## サイトの軸

- 経営工学を中心に置く
- その中に、品質管理・QMS、生産管理・オペレーション、統計・データ分析、技術士演習・統合を配置する
- 最優先は、学習しやすく、分野間のつながりが見えやすいこと
- その上で、広告収入、ポートフォリオ活用、note等の外部流入導線も考慮する

## ガイドページの標準構成

0. このテーマの位置づけ
1. 試験での出題場面
2. そのテーマの本質（よくある誤解も含む）
3. 図解またはインタラクティブ教材
4. 理解度クイズ
5. 技術士答案での使い方
6. 実務・QMS改善に向けた活用
7. 関連用語・サイト内リンク

## 学習マップページの方針

- 経営工学を中心に、品質管理・生産管理・統計・技術士演習のつながりを可視化する
- 初見ユーザーが「このサイトで何を学べるか」「次にどこへ行けばよいか」を理解できるようにする
- noteやSNSから流入したユーザーの受け皿にもする
- 将来的には、広告収入やポートフォリオ導線にも活用する

## 優先して作るコンテンツ

1. 経営工学 学習マップ
2. 技術士ケース演習ジェネレーター
3. 課題分解マトリクス
4. 答案骨子ビルダー
5. 白書アップデートボード

## 実装方針

- ガイドは MDX で管理する
- MDX本文には原則として import を書かない
- Reactコンポーネントは src/components/guide/ に置く
- クライアントコンポーネントは 'use client' を付ける
- registry.client.ts で dynamic import を一元管理する
- pages/guides/[exam]/[slug].tsx で許可コンポーネントを MDX に注入する既存方式に合わせる
- 既存コードを優先し、最小差分で進める
- 外部依存は追加しない
- 2D配列アクセスは禁止。必要な場合は safe-matrix を使う
- ローカル作業コマンドを提示する場合は PowerShell のみとする

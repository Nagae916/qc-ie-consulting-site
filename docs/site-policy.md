# n-ie-qclab サイト方針

## サイトの位置づけ

n-ie-qclab は、品質管理単独サイトでも統計単独サイトでもなく、「経営工学」を中心軸にした実務型学習サイトとする。

## 基本コンセプト

経営工学を、試験で終わらせず、現場で使える力に変える。

## サイトの軸

- 経営工学を中心に置く
- その中に、品質管理・QMS、生産管理・オペレーション、統計・データ分析、技術士演習・統合を配置する
- 最優先は、学習しやすく、分野間のつながりが見えやすいこと
- その上で、広告収入、ポートフォリオ活用、note等の外部流入導線も考慮する

## ガイドページの標準構成

0. このテーマの位置づけ
1. 試験での出題場面
2. そのテーマの本質（よくある誤解も含む）
3. 図解またはインタラクティブ教材
4. 理解度クイズ
5. 技術士答案での使い方
6. 実務・QMS改善に向けた活用
7. 関連用語・サイト内リンク

## 学習マップページの方針

- 経営工学を中心に、品質管理・生産管理・統計・技術士演習のつながりを可視化する
- 初見ユーザーが「このサイトで何を学べるか」「次にどこへ行けばよいか」を理解できるようにする
- noteやSNSから流入したユーザーの受け皿にもする
- 将来的には、広告収入やポートフォリオ導線にも活用する

## 優先して作るコンテンツ

1. 経営工学 学習マップ
2. 技術士ケース演習ジェネレーター
3. 課題分解マトリクス
4. 答案骨子ビルダー
5. 白書アップデートボード

## 実装方針

- ガイドは MDX で管理する
- MDX本文には原則として import を書かない
- Reactコンポーネントは src/components/guide/ に置く
- クライアントコンポーネントは 'use client' を付ける
- registry.client.ts で dynamic import を一元管理する
- pages/guides/[exam]/[slug].tsx で許可コンポーネントを MDX に注入する既存方式に合わせる
- 既存コードを優先し、最小差分で進める
- 外部依存は追加しない
- 2D配列アクセスは禁止。必要な場合は safe-matrix を使う
- ローカル作業コマンドを提示する場合は PowerShell のみとする

## 想定読者・ペルソナ

本サイトの中心読者は、20代後半〜40代前半の製造業技術者とする。

特に、品質管理、生産管理、技術、開発、製造、品質保証に関わり、統計検定、QC検定、生産管理オペレーション、技術士二次試験などの学習を通じて、現場改善力・QMS改善力を高めたい層を対象とする。

### 主なペルソナ

1. 若手技術者
   - QC検定・統計検定を学び、品質管理や生産管理の基礎を固めたい
   - 技術士二次試験は将来的な目標
   - 図解やインタラクティブ教材で本質を理解したい

2. 中堅技術者
   - 技術士二次試験に向けて、実務経験を課題・解決策・リスクに構造化したい
   - 経営工学、品質管理、生産管理、統計を答案に使える形で整理したい

3. 実務リーダー・管理職候補
   - QMS改善、内部監査、是正処置、品質KPI、教育訓練などを実務で改善したい
   - 若手教育や品質マネジメント改善に使える教材を探している

### デザイン・文章の方針

- UIは若手にも入りやすく、見やすく、余白を重視する
- 内容は実務者にも信頼されるよう、専門用語を正確に使う
- 公的資料・白書・法令などの一次情報を必要に応じて示す
- 文章は平易にするが、内容を浅くしない
- 「試験で解ける」だけでなく「現場で使える」ことを重視する

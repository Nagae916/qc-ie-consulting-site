# 技術士二次試験 解答事例から抽出する答案フレームルール

## 1. この資料を参照する目的

この文書は、技術士第二次試験 経営工学部門の解答事例、講座資料、添削課題、コンピテンシー整理をもとに、答案作成時に使う構造・評価観点・キーワードの使い方を内部ルール化するためのメモである。

解答本文、設問文、模範解答をそのまま転載しない。サイトでは、答案の章立て、評価されやすい観点、弱くなりやすい書き方、AnswerStructureBuilder や PastExamTrendMap に反映するルールだけを扱う。

## 2. 必須科目Ⅰの解答構成

必須科目Ⅰは、社会的・産業的背景から課題を多面的に抽出し、最重要課題を選び、解決策、施策後リスク、技術者倫理・社会の持続可能性まで一貫して述べる型で整理する。

基本ブロックは以下とする。

- 背景整理
- 多面的な課題抽出
- 最重要課題と選定理由
- 解決策
- 施策後リスクと対策
- 技術者倫理・社会の持続可能性

最重要課題は、設問前半で挙げた課題の中から選ぶ。理由は、影響範囲、根本性、緊急性、波及効果、実現可能性などで説明する。

## 3. 選択科目Ⅱ-1の解答構成

選択科目Ⅱ-1は、専門用語や手法を短く正確に説明する型で整理する。長い背景説明よりも、定義、目的、構成要素、適用場面、留意点を簡潔に並べる。

基本ブロックは以下とする。

- 定義
- 目的
- 具体例
- 手順または構成要素
- 留意点
- 活用場面

用語説明であっても、経営工学部門では QCD、品質管理、生産管理、IE、OR、統計、物流、プロジェクト管理などの実務文脈に接続する。

## 4. 選択科目Ⅱ-2の解答構成

選択科目Ⅱ-2は、実務で実施する手順、管理項目、関係者調整、効果確認を問う型として整理する。単なる知識列挙ではなく、現状把握から実施、確認、定着までの流れを示す。

基本ブロックは以下とする。

- 背景
- 現状把握
- 実施手順
- 管理項目
- 留意点
- 関係者調整
- 効果確認

手順は、調査、分析、計画、実施、評価、標準化の順に並べると読みやすい。

## 5. 選択科目Ⅲに応用できる構成

選択科目Ⅲは、専門領域の大きな課題に対して、多面的課題、最重要課題、解決策、リスク、将来展望を整理する型として扱う。必須科目Ⅰよりも専門領域の深さと将来展望を強める。

基本ブロックは以下とする。

- 背景
- 多面的課題
- 最重要課題
- 解決策
- リスク
- 将来展望
- 持続可能性

経営工学部門では、品質、生産性、物流、DX、人材、サプライチェーン、環境、リスク管理を組み合わせる。

## 6. 解答のポイントから抽出した評価観点

評価観点は以下に整理する。

- 設問要求を読み違えていない
- 問題点と課題を区別している
- 課題が多面的で、同じ内容の言い換えになっていない
- 最重要課題の選定理由が明確である
- 解決策が最重要課題に直接対応している
- 経営工学の専門語が実務文脈で使われている
- 施策後リスクが、解決策の副作用として書かれている
- 対策がリスクに直接対応している
- 技術者倫理・社会の持続可能性がテーマ固有である
- 効果確認や標準化まで含まれている

## 7. 解答事例に共通するキーワードの使い方

キーワードは列挙ではなく、課題、解決策、評価、倫理に接続して使う。

- QCD、KPI、工程能力、管理図、FMEA、CAPA、標準化は、品質・生産性・改善の根拠として使う
- IE、ECRS、ラインバランシング、ボトルネックは、生産性向上や作業改善に使う
- MES、IoT、BI、AI、データ基盤は、DX・データ活用の具体策として使う
- BCP、複数購買、在庫戦略、リードタイム短縮は、サプライチェーン強靭化に使う
- OJT、技能マップ、多能工化、ナレッジマネジメントは、人材育成・技能伝承に使う
- 公益確保、説明責任、品質不正防止、持続可能性は、倫理・社会的影響に接続する

## 8. コンピテンシーとの対応

答案フレームは、コンピテンシーを答案上で表現するための器として使う。

- 専門的学識: 手法名、適用場面、目的を正確に説明する
- 問題解決: 課題、最重要課題、解決策、リスクを一貫させる
- マネジメント: 資源、体制、QCD、KPI、進捗管理を示す
- 評価: 効果確認、KPI、レビュー、標準化を示す
- コミュニケーション: 関係者調整、情報共有、合意形成を示す
- リーダーシップ: 部門横断の推進、定着、利害調整を示す
- 技術者倫理: 公益、安全、法令順守、不正防止、説明責任を示す
- 継続研さん: 標準改定、教育訓練、技術動向の反映を示す

## 9. AnswerStructureBuilder に反映するルール

AnswerStructureBuilder には、以下を反映する。

- 必須科目Ⅰ型では、課題3つ、最重要課題、選定理由、解決策、リスク、対策、倫理・持続可能性を必須ブロックとして扱う
- リスクは「施策実施後の副作用」として入力を促す
- 倫理は「法令順守」だけでなく、テーマ固有の公益、品質不正防止、説明責任まで確認する
- Ⅱ-1型、Ⅱ-2型、Ⅲ型の答案ブロックを切り替えられるようにする
- 出力末尾に関連コンピテンシーの確認欄を表示する

## 10. PastExamTrendMap に反映するルール

PastExamTrendMap には、以下を反映する。

- 過去問を、必須Ⅰ、Ⅱ-1、Ⅱ-2、Ⅲの答案型で分類する
- 問題ごとに必要な answerBlocks を紐づける
- 問題ごとに関連コンピテンシーを表示する
- 頻出テーマと有用キーワードを、答案フレームと接続する
- 例題生成時に、過去問テーマだけでなく答案型も利用する

## 11. 例題生成MVPに反映するルール

例題生成MVPでは、以下を反映する。

- 必須科目Ⅰ型の問題では、背景、課題抽出、最重要課題、解決策、リスク、倫理を問う構成にする
- Ⅱ-1型では、用語定義、目的、構成要素、留意点を短答向けに問う
- Ⅱ-2型では、実施手順、管理項目、関係者調整、効果確認を問う
- Ⅲ型では、専門領域の課題、将来展望、持続可能性を問う
- 生成した例題には、推奨 answerBlocks と relatedCompetencies を表示する
